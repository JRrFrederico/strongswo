## O Que NÃO Foi Traduzido (E o Porquê)

As únicas coisas que não foram traduzidas foram termos que **não são seletores que você criou**, mas sim palavras-chave ou padrões da própria tecnologia web. A intenção foi preservar a funcionalidade do código.

Aqui estão os exemplos:

1.  **Tags HTML:**
    - body, header, nav, main, footer, div, p, h1, a, ul, li, etc.
    - **Motivo:** São nomes de tags fixos da linguagem HTML. Traduzir body para corpo no CSS (corpo { ... }) faria o estilo deixar de funcionar, pois não existe uma tag `<corpo>` no HTML.

2.  **Atributos data-*:**
    - [data-livro="..."], [data-capitulo="..."]
    - **Motivo:** O prefixo data- é um padrão do HTML5 para criar atributos customizados. Embora a *parte* depois do hífen pudesse ser traduzida (ex: data-livro para dado-livro), a convenção universal é manter esses atributos em inglês para facilitar a interoperabilidade com bibliotecas JavaScript e a leitura por outros desenvolvedores. Mantive-os para seguir essa boa prática.

3.  **Pseudo-classes e Pseudo-elementos CSS:**
    - :hover, :disabled, :not(), ::before, ::after
    - **Motivo:** São palavras-chave fixas da linguagem CSS e não podem ser alteradas.

4.  **Nomes de Propriedades CSS:**
    - background-color, font-size, margin-left, display, etc.
    - **Motivo:** São as propriedades da própria linguagem CSS.
